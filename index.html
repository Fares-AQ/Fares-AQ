Matrix Chain Multiplication

Interactive Lecture Tool • Algorithms I • Fares Al Qedrah • 20220002060

Lecture Notes — Start Here

1. What is the problem?

Given matrices A₁, A₂, ..., Aₙ, we want to find the best parenthesization so the total number of scalar multiplications is as small as possible.

2. Why does order matter?

Matrix multiplication is associative, so the final matrix is the same, but the scalar cost can change dramatically depending on where the chain is split.

3. Dynamic Programming insight

Large ranges are solved using smaller optimal subproblems. Since many subproblems repeat, dynamic programming stores them in a table and reuses them.

4. Tables used

We use m[i][j] for the minimum multiplication cost and s[i][j] for the best split point k for Aᵢ...Aⱼ.

5. CLO-2 angle

This algorithm does not sort matrices. Instead, it performs a search over possible split points k. This page explicitly compares full search against weaker search strategies.

6. Complexity

Dynamic programming solves Matrix Chain Multiplication in O(n³) time and O(n²) space, which is far better than brute force over all parenthesizations.
