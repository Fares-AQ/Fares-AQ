What is the problem?

Given matrices A₁…Aₙ, find the parenthesization that minimises scalar multiplications. Order matters because different groupings can have wildly different costs.

DP insight

Large ranges are solved from smaller optimal subproblems. We store results in m[i][j] (min cost) and s[i][j] (best split k) to avoid recomputing.

CLO-2 — role of search

  • The key operation is searching all valid split points k per subproblem.
  • Restricting search (first / middle / greedy) can produce suboptimal answers.
  • Use Greedy Trap preset to see a concrete counterexample.

Complexity

  • Time: O(n³) — three nested loops
  • Space: O(n²) — two n×n tables
  • Brute force: Catalan-number growth

Common mistakes

  • Wrong index in p[i-1] × p[k] × p[j]
  • Assuming left-to-right is optimal
  • Assuming greedy split is globally optimal

How to use this tool

Start with Classic + Exhaustive. Then switch to Greedy or First-split on the same input to prove weaker search gives worse results.
